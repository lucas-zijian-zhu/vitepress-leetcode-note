# [547. Number of Provinces](https://leetcode.com/problems/number-of-provinces/description/) <Badge type="warning" text="Medium" />

There are <code>n</code> cities. Some of them are connected, while some are not. If city <code>a</code> is connected directly with city <code>b</code>, and city <code>b</code> is connected directly with city <code>c</code>, then city <code>a</code> is connected indirectly with city <code>c</code>.

A **province** is a group of directly or indirectly connected cities and no other cities outside of the group.

You are given an <code>n x n</code> matrix <code>isConnected</code> where <code>isConnected[i][j] = 1</code> if the <code>i</code>-th city and the <code>j</code>-th city are directly connected, and <code>isConnected[i][j] = 0</code> otherwise.

Return the total number of **provinces**.

**Example 1:**

```
Input: isConnected = [[1,1,0],[1,1,0],[0,0,1]]
Output: 2
```

**Example 2:**

```
Input: isConnected = [[1,0,0],[0,1,0],[0,0,1]]
Output: 3
```

**Constraints:**

- <code>1 <= n <= 200</code>
- <code>n == isConnected.length</code>
- <code>n == isConnected[i].length</code>
- <code>isConnected[i][j]</code> is <code>1</code> or <code>0</code>.
- <code>isConnected[i][i] == 1</code>
- <code>isConnected[i][j] == isConnected[j][i]</code>

## 💡 Approach: DFS on Adjacency Matrix

### Intuition

Treat each city as a node in an undirected graph. The matrix tells us whether two cities are connected.  
Each time we find an unvisited city, we start DFS from it and mark all reachable cities as visited.  
That whole DFS traversal is one province.

### Algorithm

1. Create a <code>visited</code> array of size <code>n</code>.
2. Iterate through each city <code>i</code>.
3. If city <code>i</code> is not visited:
   - Increase province count.
   - Run DFS from <code>i</code>, visiting all connected cities.
4. Return the province count.

### Key Points

- This is a connected-components problem.
- The matrix representation makes neighbor scanning O(n) per node.
- DFS and BFS are both valid; this version uses DFS.

```typescript [TypeScript]
function findCircleNum(isConnected: number[][]): number {
    const n = isConnected.length;
    const visited = new Array(n).fill(false);
    let provinces = 0;

    function dfs(city: number): void {
        visited[city] = true;
        for (let next = 0; next < n; next++) {
            if (isConnected[city][next] === 1 && !visited[next]) {
                dfs(next);
            }
        }
    }

    for (let city = 0; city < n; city++) {
        if (!visited[city]) {
            provinces++;
            dfs(city);
        }
    }

    return provinces;
}
```

```java [Java]
class Solution {
    public int findCircleNum(int[][] isConnected) {
        int n = isConnected.length;
        boolean[] visited = new boolean[n];
        int provinces = 0;

        for (int city = 0; city < n; city++) {
            if (!visited[city]) {
                provinces++;
                dfs(isConnected, visited, city);
            }
        }

        return provinces;
    }

    private void dfs(int[][] isConnected, boolean[] visited, int city) {
        visited[city] = true;
        for (int next = 0; next < isConnected.length; next++) {
            if (isConnected[city][next] == 1 && !visited[next]) {
                dfs(isConnected, visited, next);
            }
        }
    }
}
```

```python [Python]
class Solution:
    def findCircleNum(self, isConnected: List[List[int]]) -> int:
        n = len(isConnected)
        visited = [False] * n
        provinces = 0

        def dfs(city: int) -> None:
            visited[city] = True
            for nxt in range(n):
                if isConnected[city][nxt] == 1 and not visited[nxt]:
                    dfs(nxt)

        for city in range(n):
            if not visited[city]:
                provinces += 1
                dfs(city)

        return provinces
```

## Complexity

- **Time Complexity:** O(n^2), because for each city we may scan a full row in the matrix.
- **Space Complexity:** O(n), for the visited array and recursion stack in the worst case.
