# [5. Longest Palindromic Substring](https://leetcode.com/problems/longest-palindromic-substring/description/?envType=study-plan-v2&envId=top-interview-150)<Badge type="warning" text="Medium" />

Given a string <code>s</code>, return the longest **palindromic** **substring** in <code>s</code>.

**Example 1:** 

```
Input: s = "babad"
Output: "bab"
Explanation: "aba" is also a valid answer.
```

**Example 2:** 

```
Input: s = "cbbd"
Output: "bb"
```

**Constraints:** 

- <code>1 <= s.length <= 1000</code>
- <code>s</code> consist of only digits and English letters.

## ðŸ’¡ Approach: Expand Around Center

### Intuition

A palindrome mirrors around its center. The key insight is:
- A palindrome can have an **odd length** (center is a single character) or **even length** (center is between two characters)
- For each possible center position, we can expand outward to find the longest palindrome
- We check both odd and even length palindromes for each position

**Key Insight:**
- Instead of checking all substrings (O(nÂ³)), we use the center expansion approach
- For each center, expand while characters match on both sides
- This reduces time complexity to O(nÂ²)

### Algorithm

1. **Iterate** through each character in the string
2. **For each position `i`, check two cases:**
   - **Odd-length palindrome**: center at `i` (expand from `i, i`)
   - **Even-length palindrome**: center between `i` and `i+1` (expand from `i, i+1`)
3. **Expand outward** from each center while `s[left] == s[right]`
4. **Track** the maximum palindrome found (start and end indices)
5. **Return** the substring from start to end

### Key Points

- Expand around center approach is more efficient than checking all substrings
- Handle both odd and even length palindromes
- Time complexity: O(nÂ²) - n centers, each expansion takes O(n)
- Space complexity: O(1) - only using variables to track indices

```kotlin [Kotlin]
class Solution {
    fun longestPalindrome(s: String): String {
        if (s.isEmpty()) return ""
        var start = 0
        var end = 0

        for (i in s.indices) {
            val len1 = expandAroundCenter(s, i, i)       // Odd length
            val len2 = expandAroundCenter(s, i, i + 1)   // Even length
            val len = maxOf(len1, len2)

            if (len > end - start) {
                start = i - (len - 1) / 2
                end = i + len / 2
            }
        }

        return s.substring(start, end + 1)
    }

    private fun expandAroundCenter(s: String, left: Int, right: Int): Int {
        var l = left
        var r = right
        while (l >= 0 && r < s.length && s[l] == s[r]) {
            l--
            r++
        }
        return r - l - 1
    }
}
```

```python [Python]
class Solution:
    def longestPalindrome(self, s: str) -> str:
        if not s:
            return ""
        start = 0
        end = 0

        for i in range(len(s)):
            len1 = self.expand_around_center(s, i, i)       # Odd length
            len2 = self.expand_around_center(s, i, i + 1)   # Even length
            length = max(len1, len2)

            if length > end - start:
                start = i - (length - 1) // 2
                end = i + length // 2

        return s[start:end + 1]

    def expand_around_center(self, s: str, left: int, right: int) -> int:
        l = left
        r = right
        while l >= 0 and r < len(s) and s[l] == s[r]:
            l -= 1
            r += 1
        return r - l - 1
```

## Complexity

- **Time Complexity:** O(nÂ²), where n is the length of the string. There are O(n) possible centers, and expanding around each center takes O(n) time in the worst case.
- **Space Complexity:** O(1), only using variables to track indices, no extra arrays or data structures.
