# [1268. Search Suggestions System](https://leetcode.com/problems/search-suggestions-system/description/)<Badge type="warning" text="Medium" />

Given an array of strings <code>products</code> and a string <code>searchWord</code>, design a system that suggests **at most three product names** after each character of <code>searchWord</code> is typed. Suggested products must have a common prefix with <code>searchWord</code>. If there are more than three, return the three **lexicographically smallest** products.

Return a list of lists of the suggested products after each character of <code>searchWord</code> is typed.

**Example 1:**

```
Input: products = ["mobile","mouse","moneypot","monitor","mousepad"], searchWord = "mouse"
Output: [
  ["mobile","moneypot","monitor"],
  ["mobile","moneypot","monitor"],
  ["mouse","mousepad"],
  ["mouse","mousepad"],
  ["mouse","mousepad"]
]
```

**Example 2:**

```
Input: products = ["havana"], searchWord = "havana"
Output: [["havana"],["havana"],["havana"],["havana"],["havana"],["havana"]]
```

**Constraints:**

- <code>1 <= products.length <= 1000</code>
- <code>1 <= products[i].length</code>, lowercase English letters
- <code>1 <= sum(products[i].length) <= 2 * 10<sup>4</sup></code>
- All <code>products[i]</code> are unique and consist of lowercase English letters.
- <code>1 <= searchWord.length <= 1000</code>, lowercase English letters.

## 💡 Approach 1: Sort + Binary Search

### Intuition

- First **sort** <code>products</code> lexicographically.
- For each prefix <code>prefix = searchWord[0..i]</code>:
  - Use binary search (<code>lower_bound</code> style) to find the first index <code>l</code> where <code>products[l] &gt;= prefix</code>.
  - All products that start with <code>prefix</code> will lie in a contiguous block starting at <code>l</code>.
  - We only need to check up to 3 elements from <code>l</code> onward and keep those that start with <code>prefix</code>.

Each prefix does one binary search plus constant-time prefix checks, so overall this is a classic **sort + binary search on strings** problem.

### Algorithm

1. Sort <code>products</code>.
2. For each prefix <code>prefix</code> of <code>searchWord</code>:
   - Binary search on <code>products</code> to find the first index <code>l</code> where <code>products[l] &gt;= prefix</code>.
   - From <code>l</code>, take at most the next 3 strings that start with <code>prefix</code>.
3. Append the collected list for each prefix to the answer.

```typescript [TypeScript]
function suggestedProducts(products: string[], searchWord: string): string[][] {
    products.sort();

    const res: string[][] = [];

    function lowerBound(target: string): number {
        let lo = 0, hi = products.length;
        while (lo < hi) {
            const mid = (lo + hi) >> 1;
            if (products[mid] < target) lo = mid + 1;
            else hi = mid;
        }
        return lo;
    }

    let prefix = "";
    for (const ch of searchWord) {
        prefix += ch;
        const start = lowerBound(prefix);
        const cur: string[] = [];
        for (let i = start; i < Math.min(start + 3, products.length); i++) {
            if (products[i].startsWith(prefix)) cur.push(products[i]);
        }
        res.push(cur);
    }
    return res;
}
```

```java [Java]
class Solution {
    public List<List<String>> suggestedProducts(String[] products, String searchWord) {
        Arrays.sort(products);
        List<List<String>> res = new ArrayList<>();

        int n = products.length;

        for (int i = 1; i <= searchWord.length(); i++) {
            String prefix = searchWord.substring(0, i);
            int l = lowerBound(products, prefix);
            List<String> cur = new ArrayList<>();
            for (int j = l; j < Math.min(l + 3, n); j++) {
                if (products[j].startsWith(prefix)) cur.add(products[j]);
            }
            res.add(cur);
        }
        return res;
    }

    private int lowerBound(String[] arr, String target) {
        int lo = 0, hi = arr.length;
        while (lo < hi) {
            int mid = (lo + hi) >>> 1;
            if (arr[mid].compareTo(target) < 0) lo = mid + 1;
            else hi = mid;
        }
        return lo;
    }
}
```

```python [Python]
class Solution:
    def suggestedProducts(self, products: List[str], searchWord: str) -> List[List[str]]:
        products.sort()
        res: List[List[str]] = []

        def lower_bound(target: str) -> int:
            lo, hi = 0, len(products)
            while lo < hi:
                mid = (lo + hi) // 2
                if products[mid] < target:
                    lo = mid + 1
                else:
                    hi = mid
            return lo

        prefix = ""
        for ch in searchWord:
            prefix += ch
            start = lower_bound(prefix)
            cur: List[str] = []
            for i in range(start, min(start + 3, len(products))):
                if products[i].startswith(prefix):
                    cur.append(products[i])
            res.append(cur)

        return res
```

## 💡 Approach 2: Trie (Alternative)

You can also use a Trie and store at each node a list of at most 3 candidate strings that share that prefix, similar to the autocomplete design in problem 642. However, this increases code size and memory usage; under this problem's constraints, **sort + binary search** is usually simpler and perfectly efficient.

---

## Complexity

- **Time:** Sort O(N log N) + for each prefix do a binary search O(|searchWord| · log N), total O(N log N + |searchWord| · log N).
- **Space:** O(1) extra space beyond the input array (we sort in-place and use only a few variables).
